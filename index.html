    Android Dev Experience

    My Journey in Android Development: Building 'BitterFitness' during my Time at Purdue University Global

    Intro

    ~~ Tackling Android development at Purdue University Global has been an intricate exploration of the Java programming language and its application in the Android ecosystem. This blog delves into the technical nuances of my college experience, specifically emphasizing my profound understanding of the Android app lifecycle, all while crafting the fitness tracking app 'BitterFitness.'


    The Android App Lifecycle Demystified:

    ~~ Understanding the Android app lifecycle was paramount in my journey. Delving into the intricacies of onCreate(), onStart(), onResume(), onPause(), onStop(), and onDestroy() methods in the context of activities and fragments was a profound learning experience. Grasping the transitions between these states enabled me to architect robust and responsive applications.


    Android Studio as the Epicenter:

    ~~ Android Studio emerged as my development fortress, offering a plethora of tools and functionalities. From the layout editor for XML design to the rich debugging features, I honed my skills to navigate seamlessly through the IDE, optimizing my coding workflow.


    Designing with Technical Precision:

    ~~ User Interface (UI) and User Experience (UX) design were not mere aesthetic pursuits but a technical challenge. Applying XML layouts, understanding the intricacies of constraint layouts, and optimizing view hierarchies were essential elements in crafting the intuitive interface of 'BitterFitness.


    Coding Gymnastics: Tackling Functionalities:

    ~~ The development of 'BitterFitness' required more than just coding prowess; it demanded a deep understanding of Android APIs and libraries. Implementing features such as sensor integration for activity tracking, local storage management, and asynchronous operations required intricate knowledge of Android SDK components.


    Real-time Syncing and Data Management:

    ~~ The synchronization of real-time data and efficient data management were technical feats in 'BitterFitness.' Leveraging Android's ContentProvider, SQLite database, and handling background tasks with AsyncTask and threads were crucial aspects of ensuring a seamless user experience.


    Model-View-Controller (MVC) Paradigm:

    ~~ A crucial aspect of architectural mastery is the adoption of a well-defined design pattern. 'BitterFitness' was constructed upon the Model-View-Controller (MVC) paradigm, a proven methodology that separates concerns and facilitates modular development. My adept application of MVC ensured a clear distinction between data management, user interface, and application logic.


    Conclusion

    ~~ In mastering app architecture during my Android development journey at Purdue University Global, 'BitterFitness' emerged as a testament to the fusion of theoretical knowledge and hands-on implementation. The strategic utilization of architectural patterns, dependency injection, reactive programming, and Jetpack components culminated in an app that not only navigates the Android app lifecycle with finesse but also sets a high standard for architectural excellence in the realm of fitness tracking applications.

Learning about programming patterns like the singleton and observer helped me programmatically regulate the game.     3. Scripting and Programming

    ~~ My journey into scripting with C# in Unity has been enjoyable. The straightforward process of scripting directly for an object, without orchestrating a complex web of commands in a single file or class, resonated well with me. The efficiency of Unity's debugger played a crucial role in identifying and rectifying mistakes. For scripting, Unity's comprehensive documentation proved to be an invaluable resource.


    4. Exploring Unity's Features

    ~~ In my exploration of Unity's features like Physics, Animation, and UI, I delved into various projects and exercises that expanded my understanding.     5. Building Your First Game

    ~~ My first game project, KitchenChaos, was an Overcooked clone. This choice allowed me to undertake the project solo and served as a valuable learning experience. Development posed challenges such as improper rigidbody placement and object clipping. However, each issue became a learning opportunity, contributing to the debugging phase.
